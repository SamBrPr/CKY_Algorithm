# CKY Parser & Grammar Transformer

This project is a Python-based implementation of the **Cocke-Younger-Kasami (CKY) algorithm**, a powerful dynamic programming approach for parsing context-free grammars. It includes tools for both deterministic and probabilistic parsing, as well as a robust transformer to ensure grammars meet the required **Chomsky Normal Form (CNF)**.

## 🚀 Features

* **Automatic CNF Conversion**: Converts standard Context-Free Grammars (CFG) into CNF by eliminating epsilon rules, unit rules, and binarizing long productions.
* **Deterministic Parsing**: Efficiently determines if a string belongs to a specific language.
* **Probabilistic Parsing (PCFG)**: Calculates the maximum probability of a string being generated by a probabilistic grammar.
* **Integrated Test Suite**: Support for JSON-based batch testing to verify grammar accuracy across multiple words.

## 🛠️ How it Works

### 1. Grammar Transformation
The CKY algorithm requires grammars to be in **Chomsky Normal Form**. The `GramTrans_CFGtoCNF` module automates this via:
1.  **Epsilon Removal**: Deleting $A \to \epsilon$ rules.
2.  **Unit Rule Removal**: Collapsing $A \to B$ rules.
3.  **Terminal Substitution**: Ensuring terminals only appear in $A \to 'a'$ format.
4.  **Binarization**: Breaking $A \to BCD$ into $A \to BX_1, X_1 \to CD$.

### 2. The CKY Algorithm
The parser builds a 2D table where each cell $(i, j)$ contains the non-terminals that can generate the substring of the input word starting at index $i$ with length $j$.
* **Level 1**: Fills the base cases (individual characters).
* **Higher Levels**: Combines previous results using the grammar rules until it reaches the start symbol `S`.

## ⚙️ Usage

### Prerequisites
* Python 3.x
* `pytokr` (required for parsing test suites)

### Running the Analyzer
1.  Place your grammar files in the `/fitxers` directory.
2.  Launch the main script:
    ```bash
    python A_programa_principal.py
    ```
